# 🚀 生产级企业合规与文档审计 Agent 落地方案

## 一、 系统核心架构设计 (System Architecture)

系统采用核心内核（内核不绑业务）+ 外置工具箱（可插拔）的解耦设计，确保极高的可扩展性。

```
                    ┌────────────────────────┐
                    │  用户上传 (PDF/图片)    │
                    └───────────┬────────────┘
                                │
                                ▼
         ┌──────────────────────────────────────────────┐
         │ 1. 多模态解析流 (Layout & Scene Graph Engine)  │ --> 解析乱序排版与表格
         └──────────────────────┬───────────────────────┘
                                │ 结构化 JSON
                                ▼
         ┌──────────────────────────────────────────────┐
         │ 2. 核心推理双智能体 (ReAct Core)              │
         │    - Agent A (Extractor 提取器)               │ <───┐
         │    - Agent B (Evaluator 评估器)               │ ├───反思反馈闭环
         └──────────────────────┬───────────────────────┘ └───┘
                                │
                  调用工具箱    ▼    注入规则
         ┌──────────────────────────────┬──────────────────────────────┐
         ▼                              ▼                              ▼
【可插拔工具箱 (Tool Registry)】  【动态规则库 (Config)】        【可视化前端 (UI)】
  - 场景 A: 简历/天眼查/竞业API   - 场景 A: HR 招聘风控规则      - Glassmorphism
  - 场景 B: 服装质检国标数据库   - 场景 B: 面料安全/合规规则       (玻璃拟态) 看板

```

### 1. 多模态前置解析引擎 (Multimodal Layout Engine)

* **任务**：解决真实世界中 PDF、Word 排版混乱、表格文字错乱的问题。
* **实现**：不直接提取纯文本。利用轻量级 Layout 算法（如 LayoutLM 或 PaddleOCR 结构化提取）识别“版面块”。针对表格和关键上下文，将其转化为**轻量级场景关系图谱（Scene Graph）**，确保两栏排版（如简历、合同）的阅读顺序绝对正确。同时对个人信息做脱敏处理

### 2. Critic-Evaluator（提取-反思）双智能体内核

为了将幻觉率降到接近 0%，拒绝“看 Vibe 输出”：

* **Agent A (Extractor)**：根据当前的规则库，调用工具链，提取出文档中的错别字、逻辑漏洞或违规风险。
* **Agent B (Evaluator)**：扮演严苛的审核员，审查 Agent A 的输出。**它执行一项硬性指标：Agent A 提出的每一个风险点，必须给出文档原件的“具体页码/段落上下文”作为证据链（Evidence）**。如果无法提供，直接驳回重做。

### 3. 可插拔工具箱 (Pluggable Tool Registry)

代码中设计统一的 `BaseTool` 基类，Agent 核心只负责根据推理调用工具，不关心工具底层的具体业务：

* **场景 A (HR/比赛)**：注入 `ResumeCheckTool`（校验时间线冲突）、`EnterpriseQueryTool`（对接天眼查 API 评估前东家竞业风险）。
* **场景 B (服装品牌面试)**：注入 `FabricStandardTool`（查询 GB 18401 等纺织品国家安全标准数据库）。

## 二、 数据化与自动化测评方案 (Evaluation Pipeline)

为了自证项目达到了“非玩具”的工业级标准，必须建立**自动化性能看板**。

### 1. 核心量化指标

* **风险召回率 (Recall >= 93%)**：文档中实际存在的合规红线/漏洞，Agent 成功漏检的概率极低。
* **审计精准率 (Precision >= 90%)**：Agent 报出的风险点是切实存在的，误报率极低。
* **证据可追溯率 (Faithfulness = 100%)**：所有输出必须 100% 带有原文出处，彻底消灭幻觉。
* **端到端时延 (Latency < 20s)**：单份复杂多页文档审计耗时稳定在 20 秒以内。

### 2. 测评工程落地 (LLM-as-a-Judge)

1. **构建黄金测试集**：手动/用 AI 构造 30 份包含“隐蔽陷阱”（如时间线重叠、存在竞业关联大厂、面料成分超标）的测试文档，并写好标准答案 `ground_truth.json`。
2. **自动化测评脚本**：每次你修改了 Agent 的 Prompt 或工具链，跑一遍测试集。调用高级商业大模型（如 GPT-4o 或 Claude 3.5）作为裁判，对比 Agent 的输出与标准答案，自动计算出当前的 **Recall** 和 **Precision**，并以结构化 JSON 记录。

## 三、 高级 UI/UX 承载 (Glassmorphism 视觉面板)

针对传统行业（服装品牌）及比赛评委，视觉冲击力是极大的加分项。

* **技术栈**：Tauri / Electron + React/Vue。
* **设计风格**：采用磨砂玻璃/高透明度玻璃拟态（Glassmorphism）设计，极简且富有科技感。
* **交互亮点**：
* **左侧**：文档（PDF/质检报告）的原件预览。
* **右侧**：Agent 的动态思考链（Thinking Process）面板。HR 或业务人员能清晰看到 Agent 正在：*“正在解析版面 -> 提取前东家 -> 发现竞业风险 -> 调用天眼查校验 -> 正在进行双体反思校验...”*。
* **下方**：一键生成结构化的 HTML/PDF 审计报告，标记出高危、中危、低危红旗点。
